# Release Notes — v0.1.0

**发布日期**：2026-07-13  
**版本**：0.1.0  
**代号**：First Tape-Out

---

## 概述

SoC Verify v0.1.0 是首个正式发布版本，覆盖 M0-M7 全部里程碑。本版本实现了从项目 Kickoff 到 Tape-Out 的 SoC 验证全流程管理能力，包括项目管理、插件系统、仿真执行、终端集成、覆盖率分析、回归测试、AI 辅助验证、环境配置、Dashboard、TO 检查清单等核心功能。

**提交统计**：21 次提交，覆盖 7 个里程碑

## 新功能

### 项目管理
- 多项目打开/关闭/切换，支持项目列表持久化
- 文件树浏览（原生 `fs.watch` 实时同步）
- CodeMirror 文件编辑器，支持 C++ / CSS / HTML / JS / JSON / Markdown / Python / YAML 等语法高亮
- 项目文件全文搜索
- 项目状态持久化与启动时自动恢复

### 插件系统
- 5 种插件类型完整实现：`case-parser` / `subsys-discoverer` / `coverage-parser` / `simulation-runner` / `sim-option-schema`
- 从 `node_modules` 或本地路径加载，支持启用/禁用
- 内置 `unisoc-subsys-discoverer` 参考实现
- 打包时自动包含 plugins 目录

### 仿真执行
- `SimulationManager` 管理仿真完整生命周期
- 编译错误解析与展示
- 仿真运行历史记录与多轮对比
- 仿真选项动态表单（基于插件 Schema 生成）
- 选项预设管理（保存/加载常用配置）
- 右键上下文菜单触发仿真

### 终端集成
- node-pty PTY + xterm.js 多标签终端
- 终端创建/写入/调整大小/销毁全生命周期管理
- 嵌入 EDA 工具命令行交互

### AI 辅助验证
- 多会话并行管理（并发上限 10）
- 流式消息传输，实时渲染
- Markdown 渲染（react-markdown + remark-gfm），支持 GFM 表格、代码块
- 工具调用卡片展示
- 会话状态机管理（创建 / 运行 / 中止 / 销毁）
- 会话 steer（中止当前推理）
- 模型切换与可用模型列表
- 技能发现：扫描项目和用户级 SKILL.md 文件
- `/` 快捷键添加技能，`@` 快捷键添加文件上下文
- 自动会话命名（基于首条消息）
- 会话持久化与恢复
- OpenAI 兼容代理，支持对接第三方 LLM 服务

### 覆盖率分析
- 多维度覆盖率：行覆盖 / Toggle 覆盖 / 功能覆盖 / 断言覆盖
- 覆盖率总览与分子系统钻取
- 覆盖率趋势可视化
- HTML / JSON 报告导出

### 回归测试
- 回归套件管理（创建 / 编辑 / 删除）
- 批量执行与结果汇总
- 回归结果历史记录

### 环境配置
- EDA 工具自动检测
- 环境变量配置向导
- 配置持久化

### Dashboard
- 项目仿真指标全景视图
- 覆盖率指标汇总
- 后台任务监控面板

### TO 检查清单
- 流片前检查项管理
- 自动评估（覆盖率阈值、回归通过率等）
- 报告导出

### 凭据管理
- API 密钥安全存储（掩码显示）
- 自定义 LLM 接口地址
- 多凭据管理

### UI / UX
- 自定义无边框窗口 + TitleBar
- 6 套主题（light / dark / midnight / carbon / nord / solarized-light）
- 三栏可调节布局（拖拽 ResizeHandle）
- 左/右栏折叠切换
- 命令面板（快捷键触发）
- Toast 通知系统
- 设置面板

### tRPC API
- 14 个子路由：`ping` / `version` / `system` / `project` / `session` / `simulation` / `terminal` / `env` / `coverage` / `regression` / `dashboard` / `to` / `settings` / `search`
- 类型安全的端到端 IPC

## 重构

- 将 omp 相关代码从 `src/main/omp/` 迁移到 `src/main/agent/` 和 `src/main/host/` 目录，职责分离更清晰
- RPC 客户端与会话启动流程重构，支持传入 API 密钥和环境变量
- Agent 客户端长任务改为发送即忘（fire-and-forget）模式，避免阻塞
- 替换 chokidar 为原生 `fs.watch`，减少依赖、提升文件监视性能

## Bug 修复

- 修复 `ELECTRON_RUN_AS_NODE` 环境变量导致 Electron 应用启动失败
- 修复会话 store 中重复渲染用户消息的问题（过滤非 assistant 角色事件）
- 修复 OpenAI 兼容协议不匹配导致的 403 错误
- 修复插件加载器在多运行环境下的路径适配问题
- 修复项目持久化数据丢失问题
- 修复项目打开时插件未自动加载的问题
- 修复会话模型设置未持久化的问题
- 修复 `endpoint` 字段重命名为 `baseUrl` 的兼容性问题
- 修复打包时 plugins 目录未正确包含的问题

## 已知限制

- 仅支持 Windows 平台打包（macOS / Linux 理论支持但未验证）
- omp 引擎以 git submodule 内嵌，不支持独立更换
- 单用户桌面应用，不提供 Web/移动端/多用户协作
- EDA 工具集成完全依赖插件，平台不内置任何 EDA 工具适配
