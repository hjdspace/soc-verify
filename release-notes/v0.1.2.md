# Release Notes — v0.1.2

**发布日期**：2026-07-16  
**版本**：0.1.2  
**上一版本**：[v0.1.1](./v0.1.1.md)

---

## 概述

SoC Verify v0.1.2 聚焦于仿真执行链路、终端集成、Diff Review、错误分析与 Unisoc 插件生态的补齐。本版本将仿真执行从后台任务迁移到终端视图，支持在终端中直接运行仿真并实时查看输出；新增 Diff Review 系统用于结构化代码 diff 审查；引入自动错误分析能力，可在仿真失败时解析日志并触发重跑；同时补齐了 Unisoc 插件集（case-parser / simulation-runner / sim-option-schema）。

**提交统计**：67 次提交（自 v0.1.1 以来）

---

## 新功能

### Diff Review 系统

- 新增 Diff Engine，支持结构化代码 diff 生成与解析
- 新增 DiffReviewView 组件，支持文件 diff 可视化审阅
- 新增 diff review 队列管理 store
- 新增 diff review tRPC procedures
- ToolCard 重构为 useCallback 模式，支持 diff 相关工具调用

### 仿真错误自动分析

- 新增 LogAnalyzer 模块，用于 EDA 仿真日志解析
- 新增 ErrorAnalysisCoordinator，统一处理仿真失败后的错误分析流程
- 仿真失败时自动调用错误分析并展示结果
- 新增 `read_file` Host Tool，支持读取文件内容
- 新增 `runsim_retry` Host Tool，支持自动重跑仿真
- 新增 `onErrorAnalysisEvent` IPC 事件回调

### 终端仿真执行

- 新增终端内仿真执行能力，仿真命令在 TerminalView 中直接运行
- 新增 RunningCasesPanel，展示运行中的仿真案例
- 新增 SimTerminalLinker，关联仿真任务与终端 session
- TerminalManager 支持数据批处理与 session 管理
- 新增 terminal simulation procedures
- 支持 `sim-running` 中心视图类型

### Unisoc 插件集

- 新增 `unisoc-case-parser` 插件，解析子系统下的验证用例
- 新增 `unisoc-simulation-runner` 插件，执行 Unisoc 仿真
- 新增 `unisoc-sim-option-schema` 插件，提供仿真选项 schema
- 新增 Rollup 与 LightningCSS 依赖用于插件构建
- CaseInfo / PluginCaseInfo 接口增强，支持更多用例属性

### Agent 运行时

- 重构 agent 运行时，支持预编译二进制 runner 模式
- `HostToolsRegistry` 支持 `cwd` 初始化选项
- 扩展已知环境变量：`PROJ_WORK`、`PROJ_ENV`
- `SimulationRunOptions` 新增 `projectRoot` 字段

## 重构

- **terminal**：重构 TerminalManager 数据批处理与 session 管理
- **simulation**：重构仿真事件处理，使用 IPC record 传递事件

- **ui**：重构 ToolCard、SubsysList、OptionDock 等组件
- **SubsysList**：支持 case tree 结构展示
- **OptionDock**：支持分组与新增图标

## 已知限制

- 仅支持 Windows 平台打包（macOS / Linux 理论支持但未验证）
- omp 引擎以 git submodule 内嵌，不支持独立更换
- 单用户桌面应用，不提供 Web/移动端/多用户协作
- EDA 工具集成完全依赖插件，平台不内置任何 EDA 工具适配
